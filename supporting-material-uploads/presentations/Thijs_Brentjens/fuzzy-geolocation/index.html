Context

In the age of analogue maps, who else than the user knew exactly what a user was looking for on a map?

Context

For many digital applications (mobile, desktop) geolocation is valuable.

Use cases with extra attention for security and privacy:

  1. Somebody wants to navigate to the doctor
  2. A police officer investigates a specific area for possible criminal activities

Sending a (very) accurate geolocation to online services may compromise privacy or may be a security risk.

Can we do better?

  • Just don't send the exact geolocation or area of interest to online services (APIs, websites, data services)
  • But then we need some smarter processing in the interaction with services

Fuzzy geolocation

Example of an interaction pattern

  • Give a map around this location

Fuzzy geolocation

Example of an interaction pattern

  • Change the center, enlarge area of interest, add random other areas, ...
  • Interact with services with the modified positions / area and random requests

Fuzzy geolocation

Example of an interaction pattern

  • Modify (filter) the response(s) to fit the original / intended area

Adding fuzziness

  • by a broker / trusted third party:
    • add fuzziness, interact with services, deliver final result to user agent
    • possibly provide additional services: caching of map resources, random requests, fake users
  • in mapping clients / user agents: smarter way of preparing the requests / interacting (built-in fuzziness for requests)
